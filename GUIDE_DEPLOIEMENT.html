🚀 Déploiement Jackbeat

Guide complet — Mise en ligne sur Render.com · Étape 7

📋 Vue d'ensemble

Le déploiement se fait en 4 grandes phases :

  1. Mettre le code sur GitHub (dépôt gratuit)
  2. Créer une base de données PostgreSQL sur Render
  3. Créer le service web Flask sur Render
  4. Configurer les variables d'environnement

Durée estimée : 20–30 minutes · Coût : Gratuit

1

Créer un dépôt GitHub

ℹ️ GitHub est une plateforme qui héberge ton code. Render va lire ton code depuis GitHub pour le déployer.

1.1 — Créer un compte GitHub (si tu n'en as pas)

Aller sur github.com → Sign up → Créer un compte gratuit.

1.2 — Créer un nouveau dépôt

  • Cliquer sur le + en haut à droite → New repository
  • Nom du dépôt : jackbeat
  • Visibilité : Public (obligatoire pour le plan gratuit Render)
  • Ne pas cocher "Add README" (on a déjà les fichiers)
  • Cliquer sur Create repository

1.3 — Envoyer le code sur GitHub

Ouvre un terminal dans le dossier jackbeat/ et exécute :

# Initialiser Git
git init
git add .
git commit -m "Initial commit — Jackbeat v1.0"

# Connecter au dépôt GitHub (remplace TON_USERNAME par ton nom GitHub)
git remote add origin https://github.com/TON_USERNAME/jackbeat.git

# Envoyer le code
git branch -M main
git push -u origin main
✅ Si Git te demande un mot de passe, utilise un Personal Access Token GitHub (Settings → Developer settings → Personal access tokens → Generate new token).
2

Créer un compte Render

Aller sur render.com → Sign Up

💡 Choisir Sign up with GitHub — cela connecte automatiquement Render à ton dépôt GitHub sans configuration supplémentaire.
  • Cliquer sur Sign up with GitHub
  • Autoriser Render à accéder à GitHub
  • Sélectionner le plan Free
3

Créer la base de données PostgreSQL

ℹ️ En développement, on utilise SQLite (un fichier local). En production sur Render, on utilise PostgreSQL (un vrai serveur de base de données).

3.1 — Créer la base

  • Dans le dashboard Render → cliquer sur New +
  • Choisir PostgreSQL
  • Remplir :
    • Name : jackbeat-db
    • Database : jackbeat
    • User : jackbeat_user
    • Region : Frankfurt (EU Central) (le plus proche d'Afrique)
    • Plan : Free
  • Cliquer sur Create Database

3.2 — Copier l'URL de connexion

Une fois la base créée, dans la page de la base de données :

  • Aller dans l'onglet Info
  • Copier la valeur External Database URL (commence par postgresql://)
  • La garder de côté — on en a besoin à l'étape 4
⚠️ Le plan Free de PostgreSQL sur Render expire après 90 jours. Pour un projet long terme, prévoir de migrer vers un plan payant (~$7/mois) ou utiliser Supabase (PostgreSQL gratuit permanent).
4

Créer le service Web (l'application)

4.1 — Créer le service

  • Dans le dashboard Render → New + → Web Service
  • Choisir Build and deploy from a Git repository
  • Sélectionner ton dépôt jackbeat
  • Cliquer sur Connect

4.2 — Configurer le service

Remplir les champs suivants :

Champ RenderValeur à entrer
Name jackbeat
Region Frankfurt (EU Central)
Branch main
Runtime Python 3
Build Command ./build.sh
Start Command gunicorn app:app --workers 2 --threads 2 --timeout 60
Plan Free
⚠️ Ne pas encore cliquer sur Create Web Service — d'abord ajouter les variables d'environnement ci-dessous.

4.3 — Variables d'environnement

Dans la section Environment Variables, ajouter ces variables :

CléValeurNote
FLASK_ENV production REQUIS
SECRET_KEY Cliquer sur Generate dans Render REQUIS
DATABASE_URL L'URL copiée à l'étape 3.2 REQUIS
ADMIN_USERNAME admin (ou ton choix) Modifiable
ADMIN_PASSWORD Un mot de passe fort de ton choix IMPORTANT
🔒 Change ADMIN_PASSWORD ! N'utilise pas jackbeat2024 en production. Choisis quelque chose de fort : ex. JB@Cam2024!

4.4 — Créer le service

Cliquer sur Create Web Service. Le déploiement démarre automatiquement.

★

Résolution des problèmes fréquents

❌ "Build failed" dans les logs

# Vérifier que build.sh a les permissions d'exécution
git update-index --chmod=+x build.sh
git commit -m "Fix: chmod build.sh"
git push

❌ "Application error" au chargement

  • Vérifier les logs Render (onglet Logs)
  • Vérifier que DATABASE_URL est bien définie
  • Vérifier que SECRET_KEY est bien définie

❌ Base de données vide (pas de graphiques)

Dans Render, aller dans Shell et exécuter :

python seed_data.py

❌ Mise à jour du code

Render redéploie automatiquement à chaque git push :

git add .
git commit -m "Mise à jour : description des changements"
git push

❌ Changer le mot de passe admin

Dans Render → Environment Variables → modifier ADMIN_PASSWORD → le service redémarre automatiquement avec le nouveau mot de passe.
